# Module Overview

dusa_tablet is built with a modular architecture that separates concerns into distinct modules. This page provides an overview of all modules defined in `fxmanifest.lua`.

## Architecture

```
dusa_tablet/
├── shared/                 # Shared between client & server
│   ├── constants.lua       # Global constants
│   ├── debug_bridge.lua    # Cross-resource logging
│   └── locales/            # Localization files
├── config/
│   └── config.lua          # Main configuration
├── server/
│   ├── debug/              # Debug & telemetry
│   ├── database/           # Database management
│   ├── apps/               # App modules (mail, news, camera, messages)
│   └── main.lua            # Server main entry
├── client/
│   ├── debug/              # Client-side debug
│   └── main.lua            # Client main entry
└── telemetry/              # Optional telemetry system
    ├── config.lua
    ├── shared/
    ├── server/
    └── client/
```

## Dependencies

The resource requires the following dependencies:

| Dependency | Purpose |
|------------|---------|
| `ox_lib` | Callbacks, locale, table utilities, keybinds |
| `dusa_bridge` | Framework-agnostic player/inventory operations |
| `oxmysql` | Database operations |

## Escrow Compatibility

All modules use the **global table pattern** for FiveM escrow compatibility:

```lua
-- Escrow compatible pattern
ModuleName = ModuleName or {}

function ModuleName.DoSomething()
    -- Implementation
end

-- No return statement at end of file
```

<Warning>
Never use `require()` in escrow files. Use global tables for cross-file communication.
</Warning>
